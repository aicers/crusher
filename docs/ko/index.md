# 개요

Crusher는 Giganto의 publish 엔드포인트에서 이벤트 데이터를 받아 일정 주기별
시계열 데이터로 집계한 뒤, 결과를 다시 Giganto의 ingest 엔드포인트로 전송하는
프로그램입니다. 실행 방식은 두 가지이며, 로컬 설정 파일을 직접 지정해 실행할
수도 있고, Manager 서버에서 설정을 받아 실행할 수도 있습니다.

## 주요 특징

<!-- markdownlint-disable MD007 MD013 -->
- 이벤트 데이터를 받아 정책 기준으로 통계 데이터를 생성합니다.
- 실행에 필요한 설정은 두 방식 중 하나로 제공됩니다.
    - 로컬 설정 모드: 사용자가 직접 작성한 TOML 설정 파일을 읽어 실행
    - 원격 설정 모드: 실행 시 Manager 서버에 접속해 Crusher 설정을 받아 실행
- 마지막 처리 시각을 저장해 재시작 후 이어서 처리할 수 있습니다.
<!-- markdownlint-enable MD007 MD013 -->

## 실행 모드

- **로컬 설정 모드**: `-c <CONFIG_PATH>`를 지정해 TOML 설정 파일을 사용
- **원격 설정 모드**: `-c` 없이 실행하고 Manager 서버에서 설정을 받아 사용

## 보안 전제(TLS)

Crusher는 QUIC/TLS 연결을 사용하며, 실행 시 아래 항목을 지정해야 합니다.

- 실행 시 `--cert`, `--key`, `--ca-certs` 인자로 인증서, 개인 키, CA
  인증서를 전달해야 합니다.

## 매뉴얼 구성

- **설치 전 준비:** 설정 파일, 인증서/키, CA 인증서 준비
- **설정:** Giganto 주소 및 저장 파일 경로 설정
- **실행:** 로컬 설정 모드와 원격 설정 모드 실행 방법
- **문제 해결:** 자주 발생하는 오류와 대응 방법입니다.

## 빠른 시작

1. `last_timestamp_data` 파일 경로 준비
2. 필요 시 `config.toml` 작성
3. 인증서, 개인 키, CA 인증서 준비
4. Crusher 실행
5. Manager 및 Giganto 서버 연결 상태 확인
6. 로그를 통해 정상 동작 여부 확인
